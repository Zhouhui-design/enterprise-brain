# 模拟排程工序表被动数据接收修复任务计划

## 任务概述
修复模拟排程工序表页面在无数据时显示404错误的问题，使其符合被动接收数据的业务逻辑。

## 任务列表

- [ ] 任务1：修改数据获取逻辑，区分空数据和错误状态
    - 1.1: 修改fetchData()函数，不将空数据视为错误
    - 1.2: 修改fetchStats()函数，优化错误处理逻辑
    - 1.3: 调整404错误的处理方式，在被动接收页面中视为正常状态

- [ ] 任务2：优化页面提示信息，提升用户体验
    - 2.1: 添加无数据状态下的友好提示组件
    - 2.2: 提供操作指引，告知用户如何触发数据推送
    - 2.3: 移除误导性的"获取数据失败"错误提示

- [ ] 任务3：调整初始化逻辑，符合被动接收设计
    - 3.1: 修改onMounted生命周期，不立即主动获取数据
    - 3.2: 保持轮询机制以监听数据更新
    - 3.3: 优化refreshData()方法的行为

- [ ] 任务4：完善错误处理机制，区分不同异常类型
    - 4.1: 区分网络错误、服务错误和正常空数据状态
    - 4.2: 完善日志记录，便于调试和监控
    - 4.3: 添加用户友好的错误提示

- [ ] 任务5：测试验证，确保修复效果
    - 5.1: 测试无数据状态下的页面显示
    - 5.2: 测试数据推送后的页面响应
    - 5.3: 测试各种异常情况的处理
    - 5.4: 验证轮询机制是否正常工作
